7チャットアプリの実装

Claude Code は Prism の API 仕様（Prism::text()->using()->withPrompt()->asText()）をソースから確認した上で、3 つのファイルを作成しました。

laravel-app/ ├── app/Http/Controllers/ChatController.php新規 ├── routes/web.php編集 └── resources/views/chat.blade.php新規

① コントローラー（Prism 呼び出しの核）

use Prism\Prism\Enums\Provider;
use Prism\Prism\Facades\Prism;

public function send(Request $request): JsonResponse
{
    $validated = $request->validate([
        'message' => ['required', 'string', 'max:4000'],
    ]);

    $response = Prism::text()
        ->using(Provider::Anthropic, config('...', env('PRISM_MODEL')))
        ->withSystemPrompt('あなたは親切で丁寧な日本語のアシスタントです。')
        ->withPrompt($validated['message'])
        ->asText();

    return response()->json(['reply' => $response->text]);
}

② ルート

Route::get('/chat', [ChatController::class, 'index'])->name('chat');
Route::post('/chat', [ChatController::class, 'send'])->name('chat.send');

③ チャット画面（Blade + 素の JS）

外部依存なしの HTML/CSS/JavaScript。fetch で POST /chat を呼び、返信を吹き出し表示します。CSRF トークンも送信。

✅ アクセス ブラウザで http://127.0.0.1:8000/chat を開くとチャット画面が表示されます。

8トラブルシュート（実際に起きた問題）

動作確認の過程で 2 つのエラーが発生し、Claude Code がログを読んで自力で原因を特定・修正しました。

問題① 500 MissingAppKeyException

原因 .env を手編集した際に APP_KEY 行が消えていた（Laravel の暗号化キーが空）。 
$ php artisan key:generate
Application key set successfully.

問題② Anthropic Error [404]: model not_found

原因 指定したモデル名 claude-3-5-haiku-latest が API に存在しなかった。

Claude Code は Anthropic の Models API を叩いて実在するモデル一覧を取得し、有効なモデルに差し替えました。

$ curl https://api.anthropic.com/v1/models -H "x-api-key: $KEY" ...
claude-opus-4-8
claude-sonnet-4-6
claude-haiku-4-5-20251001  ← これを採用（高速・低コスト）
...
✅ 成功 修正後、実際に Claude から返信を取得：
「こんにちは！AI、アシスタント、です。」

学び

  • エラーが出ても、Claude Code は storage/logs/laravel.log を読んで根本原因を突き止める。
  • 推測で済ませず、API を実際に叩いて「正しい値」を確認してから直す。

9IME（日本語変換）不具合の修正

あなたの指示

日本語を入力すると変換時でチャットに入力されてしまいます。こちらの原因特定と修正をお願いします。

原因 keydown で Enter を検知して即送信していたため、日本語の変換確定の Enter まで「送信」と誤判定していた。

修正：IME 変換中の Enter を除外

input.addEventListener('keydown', (e) => {
  // 変換確定中の Enter は無視
  if (e.isComposing || e.keyCode === 229) return;

  if (e.key === 'Enter' && !e.shiftKey) {
    e.preventDefault();
    form.requestSubmit();
  }
});
判定意味
e.isComposingIME で変換・編集中なら true（標準プロパティ）
e.keyCode === 229一部ブラウザの IME 入力中フラグ（保険）

学び

  • 日本語入力アプリの定番バグ。isComposing での除外がセオリー。
  • 「症状」を伝えるだけで、Claude Code が原因の特定から修正まで行える。
